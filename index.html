TITLE. Parallel Hash Tables. Brought to you by Manu Garg, Andrew Butko.

SUMMARY. We are going to implement concurrent hash tables that use coarse grained/fine grained locks. We are also going to implement lock-free hash tables. Finally we're going to analyze the implementations of each of them by testing them on different programs with different data access patterns.

BACKGROUND. A hash table is a data structure that maps keys to values and allows fast accesses to the data. More on hash tables:

THE CHALLENGE. The problem is challenging because lock-free data structures are hard, for instance if we implement the hash tables with separate chaining then deletion is going to be an issue. Another problem would be resizing the hash table on an insert and ensuring that is still lock-free.

RESOURCES. We will be using C++ for the coding aspects of this assignment and the GHC machines for testing/analyzing the performance of the code. We will most likely start the code from scratch but will be using ideas from research papers to supplement the implementation. looks promising.
